  • River City Rocketry

    River City Rocketry (RCR) is my university's very own student organization that builds rockets to compete in national competitions. Currently I am working on the Rocketry Telemetry Team. Here, my team and I are responsible with outfitting the rocket and deployable payload with computerized systems that will receive, transmit, and record real-time data from the launch. My personal project within the team is to create the ground station system that will be doing all of the signal reception from the rocket and payload. Once the data packets are correctly recepted from the respective source, the computer will log the data in a database. This allows engineers to not only go back and view the flight data from past launches, but also allows the telemetry team engineers to display the data on a webpage seamlessly.

  • TSP Solver and Job Scheduler

    The Travelling Salesman Problem (TSP) is a problem that is often studied in the realm of artificial intelligence. The problem entails finding the most efficient way to find the shortest/optimal path that intersects a set of two-dimensional vertices. While this seems simple for small sets, as the number of vertices increases so does the complexity of the problem. As part of the course content for my artificial intelligence class, students were assigned to develop a program throughout the semester that would solve sets of vertices. Initially the program I developed took a brute force approach, and calculated all possible combinations of vertices. However as the semester went along, students implemented multiple different approaches to solving these TSP sets including: greedy algorithms, genetic algorithms, and eventually hybrid algorithms (involving two algorithm architectures).

    As a final project for the course, I explored the Job Shop Scheduling problem and implemented a program that would help solve problem sets. The job shop scheduling problem states that a program have one job scheduler machine and a list of jobs to schedule/execute, and the program has to organize the jobs in such a way that will minimize total delay of the entire schedule. My solution implemented a hyridized algorithm that drew logic from genetic algorithms and a wisdom of the crowds algorithm.

  • Chromium Dashboards

    This project was given to me in my first few months at my co-op back in 2016. The company that I was working for had datacenters that would help manage eletronic protected health information (ePHI). When I first started with the company, they had just moved into a new and bigger office building, and so there were big new TV monitors set up all throughout the workplace. My project consisted of configuring several raspberry pis to display these dashboards on the tv that would give real time feedback on the efficiencies and inefficiencies of the datacenters and their infrastructure.

    Because I was setting up roughly 8 or so separate raspberry pis, I wanted to write a downloadable script that would streamline the process of configuring and booting up these dashboards. On top of that, I implemented a javascript file to rotate through multiple webpages, meaning that multiple data dashboards could be displayed on the same screen.
